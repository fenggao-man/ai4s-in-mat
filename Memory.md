# KG2 Memory

## 2026-04-02

### 专家审核视图优先于图谱视图

- 当前阶段的核心目标不是展示知识图谱，而是构建专家可审核、可复核、可用于评测的数据集。
- 图谱适合入库和结构化消费，但不适合作为与领域专家沟通的一线载体。
- 领域专家首先需要看懂样品、制备、测试条件、性能、表征结论和机理解释，而不是 nodes/edges。

### 当前正确工作链路

- `PDF -> structured markdown -> 样品中心标注 -> 专家审核 -> 评测基准 -> 本地模型评测 -> 知识入库`
- `structured markdown` 是当前唯一标注输入，不直接使用 PDF 做标注输入。
- 专家审核后的结果是评测基准，不是图谱本身。
- 只有当本地模型在评测中达到要求，才进入知识入库阶段。

### 当前评估原则

- 现阶段最重要的问题不是“流程是否已经自动化”，而是“自动化流程是否有效”。
- 评估必须建立在专家审核后的标准之上，否则无法判断抽取质量是否真正可用。
- 不能仅通过图谱结果判断有效性，因为专家不一定能直接读懂图谱，也难以高效指出错误来源。

### 审核导出原则

- Excel 的首要目标是让做催化剂的人能看懂、能明白、能审核。
- 专家最直白想看到的信息顺序应接近：
  1. 样品清单
  2. 制备方式
  3. 测试条件
  4. 核心性能结果
  5. 关键表征证据
  6. 机理解释
  7. 需要复核的疑点
- 导出结构应优先服务专家阅读和审核，再服务 AI 评测映射。

### 当前导出方向

- 审核 Excel 应从旧的“图谱语义视图”转向“专家审核视图”。
- `Samples` 和 `Assertions` 是当前审核主入口。
- `ReviewFlags` 用于集中处理 OCR 疑点、图文冲突、样品绑定不清等问题。
- 后续可以继续把 Excel 重排为更贴近领域专家阅读顺序的版本。
